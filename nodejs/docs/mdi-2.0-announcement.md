# 日本語組版のための Markdown 拡張「illusion Markdown (MDI) 2.0」を公開しました

日本語の小説・文芸作品を書くための Markdown 拡張フォーマット **illusion Markdown（MDI、拡張子 `.mdi`）** のバージョン 2.0 を公開しました。

MDI は、標準 Markdown（CommonMark / GFM）の記法をすべて継承したうえで、ルビ・縦中横・傍点・割注など、プレーンな Markdown では表現しにくい日本語組版の要素を、HTML タグなしの簡潔な記法で書けるようにしたフォーマットです。見た目の制御は CSS に委ね、原稿には**意味だけを書く**——という Markdown 本来の思想を、日本語組版の世界に持ち込むことを目指しています。

- 仕様書: [SYNTAX.md](https://github.com/illusions-lab/MDI/blob/main/SYNTAX.md)
- リポジトリ: [illusions-lab/MDI](https://github.com/illusions-lab/MDI)（仕様はパブリックドメイン / Unlicense）
- MDI 1.0 の仕様は [`spec/v1.0`](https://github.com/illusions-lab/MDI/tree/spec/v1.0) ブランチに保存されています

---

## 2.0 の全体像

1.0 は「ルビ・縦中横・改行制御」を中心とした最小限の拡張でした。2.0 では **書籍一冊を `.mdi` だけで組める** ことを目標に、機能を大幅に拡張しています。

| 機能 | 記法 | 2.0 での扱い |
|------|------|------|
| フロントマター | YAML | **新規** |
| ルビ（グループ／分割） | `{東京\|とうきょう}` / `{東京\|とう.きょう}` | 継承（`<rp>` フォールバック追加） |
| 縦中横 | `^12^` | 継承 |
| 傍点 | `[[em:テキスト]]` / `[[em:記号:テキスト]]` | **新規** |
| 改行抑止 | `[[no-break:テキスト]]` | 継承 |
| 段落内改行 | `[[br]]` | 継承 |
| 空白段落 | `\`（単独行） | **刷新**（旧 `[[blank]]` は互換記法として存続） |
| 割注 | `[[warichu:テキスト]]` | **新規** |
| 字間調整 | `[[kern:-0.1em:テキスト]]` | 継承 |
| 字下げ・地付き | `[[indent:N]]` / `[[bottom]]` / `[[bottom:N]]` | **新規** |
| 改ページ・改丁 | `[[pagebreak]]` / `[[pagebreak:right]]` | **新規** |
| 脚注 | `[^1]`（GFM 継承） | **新規** |

## 新機能の紹介

### フロントマター — 縦書きを「文書の属性」として宣言する

文書レベルの属性は、ファイル先頭の YAML フロントマターで宣言します。

```yaml
---
mdi: "2.0"
title: 雪女
author: 小泉八雲
lang: ja
writing-mode: vertical
---
```

縦書きか横書きかは、縦中横・傍点・脚注の描画に影響します。だからこそ MDI では、これをスタイルシートではなく**文書自身の属性**として扱います。`writing-mode: vertical` と書けば、対応レンダラーは本文を縦書き（`vertical-rl`）で組み、ページ進行方向も自動的に右→左になります。

### 傍点 — 記号は「任意の 1 文字」

日本語の強調はイタリックではなく傍点。2.0 では `[[em:...]]` 記法を導入しました。

```markdown
彼は[[em:それ]]を見た。
[[em:○:決して]]忘れない。
```

記号の指定は固定キーワードではなく、**任意の 1 文字**をそのまま書きます。既定はゴマ点（`﹅`）。`﹆` `•` `●` `○` `▲` `△` といった慣用記号はもちろん、好きな文字を傍点として使えます。

カクヨム記法の `《《テキスト》》` も互換記法として受理され、保存時に `[[em:...]]` へ正規化されます。

### 割注 — 本文の行内に二行組の注記

```markdown
その日は大安[[warichu:六曜の一つで吉日とされる]]であった。
```

伝統的な組版手法である割注（本文行内に小さく二行で組む注記）を、ようやく Markdown で書けるようにしました。

### 字下げ・地付き — 奥付や署名のために

段落の直前に単独行でマクロを置くと、直後の 1 段落だけに適用されます。

```markdown
[[indent:2]]
我輩は猫である。名前はまだ無い。

[[bottom]]
著者識

[[bottom:2]]
令和七年七月
```

`[[indent:N]]` は N 字下げ、`[[bottom]]` は地付き、`[[bottom:N]]` は地から N 字上げ。縦書きでも横書きでも同じ記法がそのまま機能します。なお、段落冒頭の慣習的な一字下げは従来どおり全角スペースで書きます——マクロは使いません。

### 改ページ・改丁

```markdown
第一章はここで終わる。

[[pagebreak]]

第二章が始まる。
```

`[[pagebreak:right]]` / `[[pagebreak:left]]` で改丁（次のページを右／左ページに指定）にも対応。EPUB・PDF・DOCX などページ概念のある出力で機能します。

### 脚注 — GFM をそのまま継承

```markdown
彼はその話を信じなかった[^1]。

[^1]: 後に事実と判明する。
```

MDI 固有の記法は導入せず、GFM / Pandoc の脚注記法をそのまま使います。EPUB 出力ではポップアップ脚注（`epub:type="footnote"`）にマップされます。

### 空白段落は `\` 一文字に

場面転換の余白などに使う「意図的な空行」は、2.0 では `\` 一文字だけの行で書きます。

```markdown
春は曙。

\
\

夏は夜。
```

`\` の行 1 つが空白段落 1 つ。1.0 の `[[blank]]` や、HTML 貼り付け由来の `<br>` 単独行も互換記法として受理され、保存時に `\` へ正規化されます。

## 仕様としての厳密化

2.0 では記法の追加だけでなく、**実装間で挙動が食い違わないための規範**を大幅に強化しました。

- **推奨（Recommended）と互換（Supported）の二層設計。** すべての機能に推奨記法を一つだけ定め、互換記法は受理したうえで保存時に正規化します。互換記法はメジャーバージョン内で削除されません。
- **文字数は書記素クラスタで数える。** 傍点記号の「1 文字」、分割ルビの親文字数、縦中横の文字数上限は、すべて拡張書記素クラスタ（Unicode UAX #29）単位。サロゲートペアの漢字も正しく 1 文字です。
- **エスケープ処理の規範化。** エスケープはすべてのインライン解析に先立って一度だけ実行され、エディタ経路とエクスポート経路で同一に動作します。
- **入れ子規則の明文化。** `[[em:{東京|とうきょう}]]`（ルビ付きテキストへの傍点）のような組み合わせが正式にサポートされ、閉じ括弧の対応規則まで仕様で定義されています。
- **ブロックマクロは常にブロック境界。** 空行を書き忘れても、`[[pagebreak]]` や `[[indent:N]]` が前の段落に吸収されることはありません。

## TXT 書き出しフレーバー — 投稿サイト・青空文庫へそのまま

2.0 仕様には、プレーンテキストへの書き出し規約 **5 フレーバー** が含まれます。

| フレーバー | 用途 |
|-----------|------|
| `plain` | ルビを捨てた素のテキスト |
| `ruby-paren` | ルビを全角括弧で表現: `漢字（かんじ）` |
| `narou` | 小説家になろう投稿用（傍点は圏点ルビで表現） |
| `kakuyomu` | カクヨム投稿用（傍点は `《《》》` 記法） |
| `aozora` | 青空文庫注記形式 |

たとえば次の MDI 原稿は——

```markdown
私は{雪女|ゆき.おんな}を見た。[[em:それ]]は確かにいた。
```

各フレーバーでこう変換されます。

```
# narou
私は｜雪《ゆき》｜女《おんな》を見た。｜そ《・》｜れ《・》は確かにいた。

# kakuyomu
私は｜雪《ゆき》｜女《おんな》を見た。《《それ》》は確かにいた。

# aozora
私は雪《ゆき》女《おんな》を見た。それ［＃「それ」に傍点］は確かにいた。
```

**原稿は `.mdi` 一つ、出力先は投稿サイトでも青空文庫でも。** 書き手がプラットフォームごとに記法を覚え直す必要はもうありません。

## エコシステム — 現在開発中

MDI 2.0 仕様と並行して、公式ツール群を開発しています。

- **[mdi-js](https://github.com/illusions-lab/mdi-js)** — Node.js 向けパーサとコンバータ群。unified / remark エコシステムの上に構築され、`.mdi` から HTML・PDF・EPUB・DOCX・TXT（5 フレーバー）への変換を提供予定です。
- **[milkdown-mdi](https://github.com/illusions-lab/milkdown-mdi)** — Milkdown エディタ向けプラグイン。MDI 記法の WYSIWYG 編集と縦書き表示を提供予定です。
- **[illusions](https://github.com/Iktahana/illusions)** — MDI をネイティブフォーマットとして採用する日本語小説執筆アプリ。

パーサとエクスポータは同一の構文定義を共有する設計になっており、「エディタでは通るのに書き出しで壊れる」という記法の分岐が構造的に起こりません。
