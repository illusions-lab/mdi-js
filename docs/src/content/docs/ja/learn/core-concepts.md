MDI の CLI、各言語バインディング、レンダラーは共通のモデルを共有する。このページでは、利用時に押さえるべき要点を説明する。

## 構文の正規実装

MDI の構文を判定するのは Rust 製の `mdi-core` だけである。JavaScript、Python、Swift などのバインディングは `mdi-core` を呼び出し、結果を各言語で扱いやすい形に変換する。独自の MDI パーサーは持たない。

そのため、ソースは断片ではなく完全な文書として渡す。たとえば `^12^` が縦中横かリテラル文字列かは、コードスパンや強調など周囲の Markdown 構造を含めて判断される。

```markdown
第^12^話       ← 縦中横
`^12^`         ← コードスパン内ではリテラル
**第^12^話**   ← 強調内でも縦中横
```

## 文書 IR

解析結果は、出力形式に依存しない木構造の**文書 IR（中間表現）**として得られる。見出し、段落、リンク、表、ルビなどは、1 つの文書ツリーに格納される。

解析結果には次の情報が含まれる。

- `syntaxVersion`: MDI 構文のバージョン
- `irVersion`: 文書 IR の形式バージョン
- `document`: 解析済みの文書ツリー
- `diagnostics`: 回復可能な問題の一覧
- `capabilities`: 解析で利用可能な機能の一覧

`syntaxVersion` と `irVersion` は別の概念である。IR を外部へ保存・送信する場合は、未知の `irVersion` を推測して処理せず、対応可否を明示的に確認する。

## ソース位置（span）

ソースに由来するノードと診断には `span` が付く。`startByte` と `endByte` は、元の UTF-8 テキストにおける半開区間（開始位置を含み、終了位置を含まない）のバイトオフセットである。

日本語文字は UTF-8 で複数バイトになる。JavaScript の UTF-16 インデックスやエディタ上の文字位置として使う場合は、表示用の位置へ変換する必要がある。

## 診断と例外

回復可能な問題は例外ではなく `diagnostics` に格納される。現在の診断コードと各フィールドは[診断とソース位置](/ja/core/diagnostics/)を参照のこと。

一方、引数の型が不正な場合や、PDF 用の Chromium が見つからない場合など、処理を継続できない問題は各 API の例外またはエラー値として返る。

## 表示設定との分離

文書 IR は原稿の意味を表す。ページサイズ、フォント、余白、字下げなどの見た目は、後段のエクスポートプロファイルで指定する。プロファイルは構文や文書の意味を変更しない。
